## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Prometheus scrape failures indicating high error rate in auth-svc.

**Root Cause:** Incorrect JWT_EXPIRY configuration in auth-svc, changed from 3600 to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY to 60.
2. Revert the configuration change to JWT_EXPIRY=3600.
3. Verify auth-svc error rate has returned to normal levels via Prometheus.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement a review process for configuration changes to auth-svc.

**Prevention:**
- Implement automated validation of configuration changes before deployment.
- Add alerting on unexpected changes to critical configuration parameters like JWT_EXPIRY.

**Affected Services:**
- auth-svc
- Any service relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
