## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** gRPC stream failures and high error rate in auth-svc.

**Root Cause:** Incorrect JWT_EXPIRY configuration in auth-svc, changed from 3600 to 60.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY to 60.
2. Roll back the JWT_EXPIRY configuration in auth-svc to 3600. Use the deployment tool (e.g., kubectl, Terraform) to revert to the previous configuration. Example: `kubectl apply -f previous_auth_svc_config.yaml`
3. Monitor auth-svc error rate and gRPC stream failures to confirm resolution.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement a code review process for configuration changes to prevent similar errors.

**Prevention:**
- Implement automated validation of configuration changes before deployment.
- Add alerting on unexpected configuration changes in critical services.
- Implement pre-production testing of configuration changes.

**Affected Services:**
- auth-svc
- Potentially any service relying on auth-svc for authentication.

**Escalation:** If not resolved in 15 minutes, page Authentication team.
