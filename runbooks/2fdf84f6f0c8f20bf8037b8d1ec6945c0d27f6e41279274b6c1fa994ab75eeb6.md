## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** High error rate in auth-svc, likely manifesting as authentication failures for users.

**Root Cause:** Incorrect JWT expiry configuration in auth-svc (JWT_EXPIRY changed from 3600 to 60).

**Immediate Fix (< 5 min):**
1. Identify the configuration change that modified JWT_EXPIRY in auth-svc.
2. Rollback the configuration change, setting JWT_EXPIRY back to 3600.
3. Monitor auth-svc error rate to confirm recovery.

**Permanent Fix:**
1. Implement a process for reviewing and approving configuration changes to critical services like auth-svc.
2. Implement automated testing for configuration changes to ensure they don't negatively impact service functionality.

**Prevention:**
- Implement pre-production testing of configuration changes.
- Add validation to the configuration management system to prevent setting JWT_EXPIRY to values below a minimum threshold (e.g., 3600).
- Improve alerting to specifically flag JWT expiry configuration changes.

**Affected Services:**
- auth-svc
- Potentially all services relying on auth-svc for authentication.

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
