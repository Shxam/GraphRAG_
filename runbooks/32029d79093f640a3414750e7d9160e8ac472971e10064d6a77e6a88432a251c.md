## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** High error rate in auth-svc, Autoscaler not triggering.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing token expiry issues.

**Immediate Fix (< 5 min):**
1. Identify the configuration change commit hash/ID in auth-svc.
2. Revert the JWT_EXPIRY configuration in auth-svc back to 3600.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Implement a configuration change review process for auth-svc.
2. Add automated testing to validate configuration changes before deployment.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes.
- Add alerting on unexpected configuration changes in critical services.

**Affected Services:**
- svc_auth_svc
- auth-svc

**Escalation:** If not resolved in 15 minutes, page Auth Team.
