## Incident Pattern: Daylight Saving Time Scheduler Failure

**Trigger:** "Daylight saving time bug" alert on scheduler service.

**Root Cause:** Configuration change related to the scheduler service incorrectly handling daylight saving time transitions.

**Immediate Fix (< 5 min):**
1. Identify recent configuration changes to the scheduler service (e.g., using deployment history tools).
2. Revert the most recent configuration change to the scheduler service.
3. Monitor scheduler and batch-jobs service for recovery.

**Permanent Fix:**
1. Analyze the reverted configuration change to identify the root cause of the DST handling issue.
2. Implement a corrected configuration change that properly handles DST transitions.

**Prevention:**
- Implement thorough testing of configuration changes, especially those related to time and scheduling, in a staging environment before deploying to production.
- Improve monitoring to specifically alert on time-related anomalies in the scheduler service.

**Affected Services:**
- scheduler
- batch-jobs

**Escalation:** If not resolved in 15 minutes, page the SRE team.
