**Trigger:** High error rate alert in auth-svc service

**Root Cause:** The JWT_EXPIRY_SECONDS config change from 3600 to 60 caused the incident.

**Immediate Fix (< 5 min):**
1. Roll back the JWT_EXPIRY_SECONDS config change to its previous value (3600) in auth-svc.
2. Verify the config change has been successfully applied.
3. Monitor the auth-svc service for a return to normal error rates.

**Permanent Fix:**
1. Review and update the JWT_EXPIRY_SECONDS config change process to prevent similar errors.
2. Implement additional monitoring and testing for config changes in auth-svc.

**Prevention:**
- Implement automated testing for config changes before deployment.
- Enhance monitoring to detect similar config-related issues before they cause incidents.
