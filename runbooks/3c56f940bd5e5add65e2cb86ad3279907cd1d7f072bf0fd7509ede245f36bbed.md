## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Message queue backlog spike

**Root Cause:** A configuration change to auth-svc reduced the JWT_EXPIRY from 3600 seconds to 60 seconds, causing increased authentication requests and a backlog.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed the JWT_EXPIRY in auth-svc.
2. Revert the configuration change to set JWT_EXPIRY back to 3600 seconds.
3. Monitor auth-svc error rate and message queue backlog for recovery.

**Permanent Fix:**
1. Investigate the reason for the initial JWT_EXPIRY change.
2. Implement proper testing and validation procedures for configuration changes to auth-svc.

**Prevention:**
- Implement a pre-production environment to test configuration changes before deploying to production.
- Add alerting on auth-svc error rate and message queue backlog.
- Implement a review process for configuration changes to critical services like auth-svc.

**Affected Services:**
- auth-svc
- unknown (Based on alert, determine the service affected by the message queue backlog)

**Escalation:** If not resolved in 15 minutes, page Authentication team.
