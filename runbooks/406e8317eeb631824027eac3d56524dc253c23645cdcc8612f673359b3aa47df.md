## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Notification service delivery failure, high error rate in auth-svc.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the configuration change commit/deployment that modified JWT_EXPIRY.
2. Roll back the JWT_EXPIRY configuration in auth-svc to the previous value of 3600.
3. Verify auth-svc error rate returns to normal.

**Permanent Fix:**
1. Investigate the reason for the incorrect JWT_EXPIRY change.
2. Implement stricter change control procedures for auth-svc configuration changes.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes, specifically for critical parameters like JWT_EXPIRY.
- Add alerting on unexpected changes to JWT_EXPIRY.

**Monitoring/alerting improvements:**
- Monitor auth-svc error rate closely.
- Implement alerting on JWT expiry related errors.

**Affected Services:**
- auth-svc
- Notification service (indirectly)

**Escalation:** If not resolved in 15 minutes, page Authentication team.
