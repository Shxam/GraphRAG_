## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** OAuth token refresh failures

**Root Cause:** Configuration change to auth-svc modified JWT_EXPIRY from 3600 to 60 seconds, causing token expiry issues.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Rollback the configuration change to JWT_EXPIRY=3600.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Implement a configuration management system with version control and audit trails for auth-svc.
2. Implement automated testing for configuration changes in auth-svc, specifically testing token expiry.

**Prevention:**
- Require peer review for all configuration changes to auth-svc.
- Implement pre-production testing of configuration changes.
- Add alerting on unexpected configuration changes in critical services.
- Implement automated validation of configuration parameters after deployment.

**Affected Services:**
- svc_auth_svc
- Any service relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page Authentication team.
