## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Object storage access denied alert; High error rate in auth-svc.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60 seconds, causing frequent token expiration and access denials.

**Immediate Fix (< 5 min):**
1. Identify the configuration change to JWT_EXPIRY in auth-svc.
2. Roll back the JWT_EXPIRY configuration in auth-svc to the previous value of 3600.
3. Monitor auth-svc error rate and object storage access to confirm resolution.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement stricter change control procedures for auth-svc configuration updates.

**Prevention:**
- Implement automated validation of configuration changes before deployment.
- Add alerting for unexpected changes to critical configuration parameters like JWT_EXPIRY.
- Implement pre-production testing of configuration changes.

**Affected Services:**
- auth-svc
- Services relying on auth-svc for authentication (unknown, but likely multiple)

**Escalation:** If not resolved in 15 minutes, page Authentication team.
