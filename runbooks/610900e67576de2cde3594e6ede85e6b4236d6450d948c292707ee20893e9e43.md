## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** API version deprecation cascade alert, high error rate in auth-svc.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 seconds to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed the JWT_EXPIRY value in auth-svc.
2. Revert the configuration change, setting JWT_EXPIRY back to 3600 seconds.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement stricter change control procedures for auth-svc configuration.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes, specifically for critical parameters like JWT_EXPIRY.
- Add alerting on unexpected configuration changes for critical services.

**Affected Services:**
- auth-svc
- Any service relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page Authentication team.
