## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** GraphQL query timeout

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing frequent token expiration and high error rate.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Rollback the configuration change to set JWT_EXPIRY back to 3600.
3. Monitor auth-svc error rate and GraphQL query response times to confirm resolution.

**Permanent Fix:**
1. Implement a configuration change review process for auth-svc.
2. Document the correct JWT_EXPIRY value and its impact on other services.

**Prevention:**
- Implement automated validation for configuration changes to prevent invalid values.
- Add alerting for unexpected changes to critical configuration parameters like JWT_EXPIRY.
- Implement pre-production testing of configuration changes.

**Affected Services:**
- auth-svc
- GraphQL API (due to dependency on auth-svc)

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
