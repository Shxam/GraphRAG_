## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Container image pull failures (caused by authentication failures due to short JWT expiry)

**Root Cause:** The JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 seconds to 60 seconds, causing frequent token expirations and subsequent authentication failures.

**Immediate Fix (< 5 min):**
1. Identify the deployment configuration for auth-svc (e.g., Kubernetes Deployment, configuration file).
2. Revert the JWT_EXPIRY configuration value in auth-svc back to 3600 seconds.
3. Redeploy auth-svc with the corrected configuration.

**Permanent Fix:**
1. Implement a robust configuration management system with version control and change tracking for auth-svc.
2. Establish a peer review process for all configuration changes to auth-svc before deployment.

**Prevention:**
- Implement automated validation of configuration changes before deployment to prevent incorrect values.
- Add monitoring and alerting for JWT expiry failures in auth-svc.

**Affected Services:**
- auth-svc
- All services dependent on auth-svc for authentication.

**Escalation:** If not resolved in 15 minutes, page Authentication team.
