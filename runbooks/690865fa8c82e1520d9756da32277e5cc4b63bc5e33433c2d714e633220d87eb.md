## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** API key rotation failure, high error rate in auth-svc.

**Root Cause:** The JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 seconds to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the commit or deployment that changed the JWT_EXPIRY configuration in auth-svc.
2. Revert the JWT_EXPIRY configuration back to 3600 seconds.
3. Verify the auth-svc service is healthy and the error rate has decreased.

**Permanent Fix:**
1. Investigate the process that allowed the incorrect configuration change to be deployed.
2. Implement stricter validation and testing of configuration changes before deployment.

**Prevention:**
- Implement pre-deployment checks to validate configuration values against expected ranges.
- Add alerting on unexpected configuration changes in critical services.

**Affected Services:**
- auth-svc
- Potentially all services relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
