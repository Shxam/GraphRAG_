## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Billing calculation errors, high error rate in auth-svc.

**Root Cause:** A configuration change to auth-svc modified the JWT_EXPIRY from 3600 to 60 seconds, causing token expiry issues.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Revert the JWT_EXPIRY configuration in auth-svc back to 3600.
3. Monitor auth-svc error rate and billing calculations to confirm resolution.

**Permanent Fix:**
1. Implement a robust configuration management system with version control and audit trails for auth-svc.
2. Establish a peer review process for all configuration changes to auth-svc, especially those related to security settings.

**Prevention:**
- Implement automated testing for configuration changes, specifically validating JWT expiry settings.
- Add monitoring for JWT expiry duration and unexpected changes in auth-svc configuration.
- Implement alerting on sudden increases in auth-svc error rates.

**Affected Services:**
- auth-svc
- Billing Service (indirectly)

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
