## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** High error rate in auth-svc service.

**Root Cause:** Incorrect JWT_EXPIRY configuration in auth-svc, changed from 3600 to 60.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Revert the JWT_EXPIRY configuration in auth-svc back to 3600.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Implement a configuration management system with version control and rollback capabilities for auth-svc.
2. Implement automated testing to validate configuration changes before deployment.

**Prevention:**
- Require peer review for all configuration changes to auth-svc.
- Implement pre-production testing of configuration changes.
- Add alerting for unexpected configuration changes.

**Affected Services:**
- auth-svc
- Potentially all services relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page Authentication team.
