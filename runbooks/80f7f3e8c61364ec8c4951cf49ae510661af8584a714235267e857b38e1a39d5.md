## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Payment processing failures indicate authentication issues potentially stemming from JWT expiry.

**Root Cause:** A configuration change to auth-svc modified the JWT_EXPIRY from 3600 seconds to 60 seconds, causing authentication failures.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Revert the configuration change, setting JWT_EXPIRY back to 3600.
3. Monitor auth-svc error rate and payment processing success rate.

**Permanent Fix:**
1. Implement a robust configuration change management process for auth-svc, including peer review and automated testing.
2. Investigate why the incorrect JWT_EXPIRY value was deployed.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes in auth-svc.
- Add alerting on unexpected changes to critical configuration parameters like JWT_EXPIRY.
- Implement automated testing to verify authentication flow with different JWT expiry values.

**Affected Services:**
- auth-svc
- Payment Processing Service (inferred from alert)

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
