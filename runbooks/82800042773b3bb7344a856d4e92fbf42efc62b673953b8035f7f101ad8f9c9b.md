## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Microservice mesh timeout storm, high error rate in auth-svc.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed the JWT_EXPIRY value in auth-svc.
2. Rollback the auth-svc configuration to the previous working state (JWT_EXPIRY = 3600).
3. Verify auth-svc error rate has returned to normal.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement a review process for configuration changes to critical services like auth-svc.

**Prevention:**
- Implement automated validation for configuration changes to prevent invalid values.
- Add alerting on unexpected configuration changes to critical services.
- Improve documentation on critical configuration parameters and their impact.

**Affected Services:**
- auth-svc
- Potentially all services relying on auth-svc for authentication.

**Escalation:** If not resolved in 15 minutes, page Authentication team.
