## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Ingress controller 502 errors

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing token expiry issues.

**Immediate Fix (< 5 min):**
1. Identify the auth-svc deployment/configuration file (e.g., Kubernetes Deployment YAML, environment variable configuration).
2. Revert the JWT_EXPIRY configuration in auth-svc back to 3600.
3. Redeploy/restart auth-svc to apply the configuration change.

**Permanent Fix:**
1. Investigate the process that led to the incorrect configuration change.
2. Implement stricter change control procedures for auth-svc configuration changes.

**Prevention:**
- Implement a peer review process for all configuration changes to auth-svc.
- Add automated validation to the configuration deployment process to check for valid JWT_EXPIRY values.
- Implement monitoring for JWT expiry related errors in auth-svc.

**Affected Services:**
- auth-svc
- All services relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
