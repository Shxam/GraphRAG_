## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Audit log corruption alert and high error rate in auth-svc.

**Root Cause:** Incorrect JWT_EXPIRY configuration in auth-svc, changed from 3600 to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the user who made the JWT_EXPIRY configuration change in auth-svc.
2. Rollback the JWT_EXPIRY configuration in auth-svc to the previous value of 3600.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Implement a change management process for configuration changes in auth-svc.
2. Review and update the auth-svc configuration documentation to clearly define the purpose and impact of JWT_EXPIRY.

**Prevention:**
- Implement pre-production testing of configuration changes.
- Implement automated validation of configuration changes before deployment.
- Add alerting on unexpected configuration changes to critical parameters like JWT_EXPIRY.

**Affected Services:**
- auth-svc
- Potentially all services relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page Authentication team.
