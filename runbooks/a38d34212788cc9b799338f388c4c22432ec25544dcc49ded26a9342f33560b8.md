## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Search service degraded accuracy. High error rate in auth-svc.

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing token expiration issues.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Rollback the JWT_EXPIRY configuration in auth-svc to the previous value of 3600.
3. Monitor auth-svc error rate to confirm recovery.

**Permanent Fix:**
1. Investigate the reason for the incorrect configuration change.
2. Implement a review process for configuration changes to auth-svc.

**Prevention:**
- Implement automated validation of configuration changes before deployment.
- Add alerting on JWT expiry related metrics in auth-svc.

**Affected Services:**
- auth-svc
- Search service

**Escalation:** If not resolved in 15 minutes, page the Auth team.
