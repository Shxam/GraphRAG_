## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** High error rate in auth-svc, likely related to authentication failures.

**Root Cause:** Incorrect JWT_EXPIRY configuration in auth-svc, changed from 3600 to 60.

**Immediate Fix (< 5 min):**
1. Identify the current JWT_EXPIRY value in auth-svc configuration.
2. Revert the JWT_EXPIRY configuration in auth-svc to 3600.
3. Restart the auth-svc service to apply the configuration change.

**Permanent Fix:**
1. Investigate the reason for the JWT_EXPIRY change.
2. Implement a configuration change management process with peer review and testing.

**Prevention:**
- Implement automated validation of configuration changes before deployment.
- Add monitoring for JWT expiry related errors in auth-svc.

**Affected Services:**
- auth-svc
- Any service relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page the Authentication team.
