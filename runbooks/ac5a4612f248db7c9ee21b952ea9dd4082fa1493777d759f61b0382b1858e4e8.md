## Incident Pattern: Auth Service JWT Expiry Configuration Error

**Trigger:** Auth service high latency alert.

**Root Cause:** The JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing frequent token expirations and increased authentication requests.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed the JWT_EXPIRY value in auth-svc.
2. Revert the configuration change, setting JWT_EXPIRY back to 3600.
3. Redeploy auth-svc with the corrected configuration.

**Permanent Fix:**
1. Implement a robust configuration management system with version control and change tracking for auth-svc.
2. Implement automated testing to validate configuration changes before deployment.

**Prevention:**
- Implement peer review process for all configuration changes to auth-svc.
- Add pre-deployment checks to validate JWT_EXPIRY value.
- Implement alerting on configuration changes to critical parameters like JWT_EXPIRY.

**Affected Services:**
- auth-svc
- unknown (dependent services)

**Escalation:** If not resolved in 15 minutes, page Authentication team.
