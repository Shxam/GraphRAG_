## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Secrets manager access failure alert, high error rate in auth-svc.

**Root Cause:** Configuration change to auth-svc modified JWT_EXPIRY from 3600 to 60 seconds.

**Immediate Fix (< 5 min):**
1. Identify the configuration change to auth-svc related to JWT_EXPIRY.
2. Roll back the JWT_EXPIRY configuration in auth-svc to the previous value of 3600.
3. Monitor auth-svc error rate to confirm resolution.

**Permanent Fix:**
1. Implement a robust configuration change management process for auth-svc.
2. Review and update documentation for auth-svc configuration parameters, specifically JWT_EXPIRY.

**Prevention:**
- Implement pre-production testing of configuration changes to auth-svc.
- Add validation to the configuration update process to prevent invalid values for JWT_EXPIRY.
- Implement alerting on unexpected configuration changes to critical services.

**Affected Services:**
- auth-svc

**Escalation:** If not resolved in 15 minutes, page Authentication team.
