## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** High error rate on `auth-svc` service, indicating a potential configuration issue.

**Root Cause:** The `JWT_EXPIRY_SECONDS` configuration was changed from 3600 to 60, causing authentication issues.

**Immediate Fix (< 5 min):**
1. Roll back the `JWT_EXPIRY_SECONDS` config change to its previous value of 3600.
2. Verify the `auth-svc` service configuration to ensure the change was successfully reverted.
3. Monitor the `auth-svc` service for a return to normal operation.

**Permanent Fix:**
1. Review and update the configuration management process to prevent similar changes in the future.
2. Implement additional monitoring and alerting for `auth-svc` service configuration changes.

**Prevention:**
- Implement automated testing for configuration changes to catch potential issues before deployment.
- Enhance monitoring and alerting for `auth-svc` service errors to detect similar issues earlier.

**Affected Services:**

- `api-gateway` (potentially affected, but not directly linked to the causal chain)

**Escalation:** If not resolved in 30 minutes, page Payments Team.
