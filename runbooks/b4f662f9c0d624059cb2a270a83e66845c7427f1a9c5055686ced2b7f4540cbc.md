## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** CDN cache poisoning detected

**Root Cause:** JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing token invalidation and CDN caching of error responses.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Revert the JWT_EXPIRY configuration in auth-svc back to 3600.
3. Redeploy auth-svc with the corrected configuration.

**Permanent Fix:**
1. Implement a robust configuration management system with version control and rollback capabilities for auth-svc.
2. Implement automated testing to validate configuration changes before deployment, specifically testing token expiry.

**Prevention:**
- Implement peer review for all configuration changes to auth-svc.
- Implement pre-production testing of configuration changes.
- Add alerting on auth-svc error rate increases.

**Affected Services:**
- auth-svc
- CDN

**Escalation:** If not resolved in 15 minutes, page Authentication team.
