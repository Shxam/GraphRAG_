## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** WebSocket connection drops

**Root Cause:** JWT_EXPIRY in auth-svc was incorrectly configured to 60 seconds instead of 3600 seconds.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc.
2. Rollback the configuration change to set JWT_EXPIRY back to 3600 seconds.
3. Monitor auth-svc error rate and WebSocket connection stability.

**Permanent Fix:**
1. Review the configuration change process for auth-svc.
2. Implement automated testing for configuration changes to prevent similar issues.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes, specifically for critical parameters like JWT_EXPIRY.
- Add alerting on unexpected configuration changes to critical services.

**Affected Services:**
- auth-svc
- Any service relying on auth-svc for authentication

**Escalation:** If not resolved in 15 minutes, page Authentication team.
