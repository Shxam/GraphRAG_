**Trigger:** High 5xx errors in auth-svc alert

**Root Cause:** The JWT_EXPIRY_SECONDS configuration was changed from 3600 to 60, causing authentication errors.

**Immediate Fix (< 5 min):**
1. Roll back the JWT_EXPIRY_SECONDS config change to its previous value (3600) in auth-svc.
2. Verify the config change has been applied successfully.
3. Monitor the auth-svc error rates for improvement.

**Permanent Fix:**
1. Review and update the configuration management process to prevent similar changes in the future.
2. Implement additional monitoring and alerts for JWT expiry configuration changes.

**Prevention:**
