## Incident Pattern: JWT Expiry Configuration Error

**Trigger:** Health check cascade failure

**Root Cause:** The JWT_EXPIRY configuration in auth-svc was incorrectly changed from 3600 to 60, causing token expiry issues and a high error rate.

**Immediate Fix (< 5 min):**
1. Identify the commit/deployment that changed JWT_EXPIRY in auth-svc to 60.
2. Rollback the configuration change to JWT_EXPIRY=3600 in auth-svc.
3. Monitor auth-svc health checks and error rates to confirm recovery.

**Permanent Fix:**
1. Investigate the process that allowed the incorrect JWT_EXPIRY configuration to be deployed.
2. Implement stricter validation and testing for configuration changes in auth-svc.

**Prevention:**
- Implement pre-deployment validation checks for configuration changes, specifically for critical parameters like JWT_EXPIRY.
- Add alerting on unexpected changes to critical configuration parameters.

**Affected Services:**
- auth-svc
- [Any service dependent on auth-svc for authentication]

**Escalation:** If not resolved in 15 minutes, page Authentication team.
